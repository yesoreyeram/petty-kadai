# One-tool-per-week plan (Go backend, Next.js frontend)

This plan introduces exactly one new tool, concept, or microservice per week to keep cognitive load low. Each week folder contains:

- tools-introduced
- concepts-covered
- proposed-architecture (Mermaid diagram)
- changes-to-system-design
- tasks-checklist (actionable and testable)
- skills-required and prerequisites
- deliverables and acceptance-criteria
